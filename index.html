Welcome!

This is Coderashed. Too long have blogs taught how to write code! Join me as we instead step through the software discovery process. In doing so we will discover how the software wants to be written instead of how we want to write it!

Goal Outline

Beginning first with a step that most unfortunately skip, we will gather an understanding of the interface that we are working with, in this case PHP, and how it interacts with the various interfaces of the operating system. From there, we will gather an understanding of what it is our software actually has to accomplish. We will then develop the unit tests that will be the proofs of our application and tie them into a build package for all revisions. Finally, we will begin developing our code and running it against our previously defined unit tests.

We will begin with a core system. With that in place we will proceed to build various sub-systems that will work on top of it. What we will end up with is a pseudo-bulletproof extremely flexible application that works exactly as we would expect. As we make modifications, we will be able to regression test against previous builds by once again running the unit tests against the build.

I hope that this blog is enlightening, and please feel free to send comments and suggestions.
